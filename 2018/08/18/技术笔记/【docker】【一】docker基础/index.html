docker 最佳实践

  1. 精简镜像用途
  2. 选用合适的基础镜像
  3. 提供足够清晰的命令注释和维护者信息
  4. 正确的使用版本号
  5. 减少镜像层数
  6. 及时删除临时文件和缓存文件
  7. 提高生产速度
  8. 调整合理的指令顺序
  9. 减少外部源的干扰
